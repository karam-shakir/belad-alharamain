## 3. طريقة التعديل (3 خطوات)

### الخطوة 1️⃣ — اضغط أيقونة القلم
في أعلى يمين الملف ستجد أيقونة قلم ✏️ (Edit this file). اضغطها.

### الخطوة 2️⃣ — عدّل ما تريد
سيتحوّل الملف إلى محرّر نصوص. عدّل ما بين علامات الاقتباس `'...'` فقط.

### الخطوة 3️⃣ — احفظ التغييرات
- انزل لأسفل الصفحة
- ستجد صندوق **Commit changes** أخضر اللون
- اكتب وصفاً مختصراً لما عدّلته (اختياري) — مثلاً: "تحديث رقم الجوال"
- اضغط **Commit changes** → ثم **Commit changes** مرة ثانية في النافذة المنبثقة

✅ **انتهى!** خلال **1-2 دقيقة** سيظهر التحديث على الموقع تلقائياً.

## 4. أمثلة عملية

### مثال 1: تغيير رقم الجوال

**ابحث عن هذا الجزء في الملف:**
```ts
export const contact = {
  phone:    '+966 50 000 0000',     // ← هنا
  whatsapp: '966500000000',
```

**عدّل الرقم بين علامتي الاقتباس فقط:**
```ts
  phone:    '+966 55 123 4567',     // ✅ تم التعديل
```

---

### مثال 2: تغيير رقم الواتساب

> ⚠️ **مهم:** الواتساب يُكتب **بدون** علامة `+` أو فراغات أو شرطات.

```ts

```

---

### مثال 3: تحديث رابط حساب X (تويتر)


**بعد التعديل:**
```ts
  twitter: 'https://x.com/beladalharamain',
```

### مثال 7: حذف خدمة من قائمة الخدمات

**قبل (4 خدمات):**
```ts
export const services = [
  { icon: 'fa-kaaba',  ... title: { ar: 'حج الداخل', ...} },
  { icon: 'fa-moon',   ... title: { ar: 'العمرة', ...} },
```

**بعد (3 خدمات):** احذف السطر بالكامل بما فيه الفاصلة في نهايته.
```ts
export const services = [
  { icon: 'fa-kaaba',  ... title: { ar: 'حج الداخل', ...} },
  { icon: 'fa-moon',   ... title: { ar: 'العمرة', ...} },
  { icon: 'fa-bus',    ... title: { ar: 'خدمات النقل', ...} },

```

### مثال 8: إضافة خدمة جديدة

انسخ سطر خدمة موجودة بالكامل، والصقه قبل القوس `]` في نهاية المصفوفة، ثم عدّل القيم.

```ts
export const services = [
  // الخدمات الموجودة...
  { icon: 'fa-bus', color: 'from-teal-dark to-teal', accent: 'teal',
    title: { ar: 'خدمات النقل', en: 'Transport' },
    body:  { ar: '...', en: '...' } },

  // ✅ خدمة جديدة (الصقها هنا)
  { icon: 'fa-utensils', color: 'from-gold-dark to-gold', accent: 'gold',
    title: { ar: 'الإعاشة', en: 'Catering' },
    body:  { ar: 'وجبات حلال متنوعة طوال الرحلة.',
             en: 'Diverse halal meals throughout the trip.' } },
];
```

> 💡 **أيقونات Font Awesome:** ابحث عن الأيقونة المناسبة في https://fontawesome.com/icons (انسخ اسمها مع البادئة `fa-`).

## 5. كيف تُضيف صورة جديدة

### مثال: إضافة صورة لقسم "من نحن"

#### الخطوة 1: ارفع الصورة إلى GitHub
1. افتح: https://github.com/karam-shakir/belad-alharamain/tree/main/public/images
2. اضغط زر **Add file** → **Upload files**
3. اختر الصورة من جهازك (يفضّل بصيغة `.jpg` أو `.png`)
4. اكتب اسماً واضحاً للصورة (مثلاً: `about-team.jpg`)
5. اضغط **Commit changes**

#### الخطوة 2: اربط الصورة في ملف المحتوى
افتح `site.ts` وعدّل قسم `about`:

```ts
export const about = {
  description: { ar: '...', en: '...' },
  
  image: '/images/about-team.jpg',     // ✅ هنا اكتب مسار الصورة
  
  experienceBadge: { ar: '+15 سنة خبرة', en: '+15 Years' },
  ...
};
```

> ⚠️ المسار **يبدأ دائماً** بـ `/images/` ثم اسم الصورة.

## 6. كيف تُضيف فيديو يوتيوب

### الخطوة 1: استخرج معرّف الفيديو (Video ID)
من الرابط: `https://www.youtube.com/watch?v=dQw4w9WgXcQ`  
معرّف الفيديو هو: **`dQw4w9WgXcQ`** (ما بعد `v=`)

من الرابط المختصر: `https://youtu.be/dQw4w9WgXcQ`  
معرّف الفيديو هو: **`dQw4w9WgXcQ`** (ما بعد آخر `/`)

### الخطوة 2: استبدل المعرّفات الافتراضية
في `site.ts` ابحث عن قسم `videos` واستبدل `VIDEO_ID_1`، `VIDEO_ID_2` ... إلخ:

```ts
export const videos = [
  { youtubeId: 'dQw4w9WgXcQ',   // ✅ ضع المعرّف الحقيقي هنا
    title:    { ar: 'موسم الحج 1445 هـ', en: 'Hajj Season 1445 AH' },
    subtitle: { ar: 'لقطات مميزة', en: 'Highlights' } },
  ...
];
```

## 7. كيف تُغيّر شعار الشركة

#### الخطوة 1: ارفع الشعار الجديد
1. افتح: https://github.com/karam-shakir/belad-alharamain/tree/main/public/images
2. اضغط على ملف `logo.png` الموجود
3. اضغط زر **Delete this file** ثم Commit
4. ارجع للمجلد، اضغط **Add file** → **Upload files**
5. ارفع الشعار الجديد **باسم `logo.png` تماماً**
6. اضغط **Commit changes**

> ✅ ينتشر الشعار تلقائياً في الموقع كله (Header، Hero، Footer).

## 8. أخطاء شائعة وكيف تتجنبها

### ❌ خطأ: حذف فاصلة بطريق الخطأ
```ts
// خطأ:
title: { ar: 'العنوان' en: 'Title' }     // ✗ ناقص فاصلة
// صحيح:
title: { ar: 'العنوان', en: 'Title' }    // ✓
```

### ❌ خطأ: حذف علامة اقتباس
```ts
// خطأ:
phone: +966 50 000 0000,                  // ✗ بدون اقتباس
// صحيح:
phone: '+966 50 000 0000',                // ✓
```

### ❌ خطأ: استخدام علامات اقتباس خاطئة
```ts
// خطأ:
phone: "+966 50 000 0000”,                // ✗ علامات منحنية ”…“
// صحيح:
phone: '+966 50 000 0000',                // ✓ علامات مستقيمة '…'
```

### ❌ خطأ: تعديل اسم الحقل بدلاً من القيمة
```ts
// خطأ (غيّرت اسم الحقل):
phoneNumber: '+966 50 000 0000',          // ✗
// صحيح (احتفظ باسم الحقل، عدّل القيمة فقط):
phone: '+966 50 000 0000',                // ✓
```

### ❌ خطأ: نسيان إغلاق قوس
```ts
// خطأ:
mvv: [
  { ... },
  { ...                                    // ✗ ناقص } في النهاية
];
```

### 🛡️ كيف تتأكد قبل الحفظ
GitHub يعرض الكود مع **ألوان**:
- إذا كان النص بلون **برتقالي/أخضر** متناسق = الكود سليم
- إذا ظهر **أحمر** أو فقد التلوين فجأة = هناك خطأ في تلك المنطقة

## 9. استعادة نسخة سابقة (إذا أخطأت)

إن لاحظت أن الموقع توقف عن العمل بعد تعديلك:

1. افتح: https://github.com/karam-shakir/belad-alharamain/commits/main
2. ستجد قائمة بكل التعديلات السابقة (Commits)
3. اختر آخر تعديل **سليم** (قبل الذي تسبّب بالمشكلة)
4. اضغط على رمز `<>` بجانبه (Browse repository at this point)
5. أو ببساطة: اضغط على عنوان الـ Commit، ثم اضغط **Revert** أعلى يمين الصفحة
6. اكتب رسالة وصف، ثم Commit → سيستعيد الموقع لتلك النقطة

> 💡 **لا داعي للقلق** — كل تعديل محفوظ في GitHub ويمكن التراجع عنه دائماً.

## 10. التحقق من ظهور التحديث

بعد كل Commit:

1. **انتظر ~1-2 دقيقة** (Vercel يحتاج وقتاً لإعادة البناء)
2. افتح الموقع: https://belad-alharamain-2aeu.vercel.app
3. اعمل **تحديث قسري** للصفحة:
   - **Windows/Linux:** `Ctrl + F5` أو `Ctrl + Shift + R`
   - **Mac:** `Cmd + Shift + R`
   - **جوال:** اسحب الصفحة من الأعلى للأسفل، أو امسح كاش المتصفح من الإعدادات

### كيف تتأكد أن Vercel انتهى من النشر؟

اذهب إلى: https://vercel.com/dashboard → اختر المشروع → ستجد آخر Deployment.
- 🟢 **Ready** = جاهز ومنشور
- 🟡 **Building** = ما زال يعمل، انتظر دقيقة
- 🔴 **Error** = حدث خطأ في البناء (راجع الأخطاء الشائعة في القسم 8)

## 🆘 ماذا تفعل لو لم يعمل شيء؟

1. **تحقق من حالة Vercel:** قد يكون البناء فشل بسبب خطأ في الكود.
2. **افتح Console المتصفح:** `F12` → تبويب Console — ستظهر أي رسائل خطأ.
3. **استعد آخر نسخة سليمة** (انظر القسم 9).
4. **تواصل مع المطوّر** إذا استمرّت المشكلة.
